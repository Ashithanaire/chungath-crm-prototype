1. SLDS OVERVIEW & COMPLIANCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Salesforce Lightning Design System (SLDS) is the official design system for Salesforce.
The prototype strictly adheres to SLDS guidelines for:
├─ Visual Design (colors, typography, spacing)
├─ Component Patterns (buttons, cards, modals, tables)
├─ Layout Patterns (page layouts, responsive design)
├─ Interaction Patterns (navigation, workflows, states)
└─ Accessibility Standards (keyboard navigation, screen readers)

Why SLDS Matters:
├─ Consistency: Users familiar with Salesforce will immediately understand the app
├─ Familiarity: Follows official Salesforce UI patterns
├─ Compliance: Meets Salesforce accessibility standards
├─ Professional: Enterprise-grade design language
├─ Efficiency: Rapid development using Salesforce components
└─ Support: Official documentation & community resources available

This prototype uses SLDS core principles & components to ensure
a seamless Lightning Experience for end users.

4. SPACING & LAYOUT (SLDS GRID SYSTEM)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Space Scale:
├─ 4px: Minimal spacing (icon-to-text, tight elements)
├─ 8px: Small spacing (form field gaps, icon spacing)
├─ 12px: Standard spacing (table cell padding, form row gaps)
├─ 16px: Medium spacing (card padding, section gaps, button padding)
├─ 20px: Large spacing (card titles, major sections)
├─ 24px: Extra large spacing (page padding, container margins)
└─ 32px: Page-level spacing (vertical section separators)

Application in Prototype:

Component Padding:
├─ Card: 20px padding (internal content area)
├─ Card Title: margin-bottom 16px
├─ Table Cell: 12px padding (vertical & horizontal)
├─ Form Field: 12px gap between fields
├─ Button: 10px padding (vertical), 16px (horizontal)
└─ Modal: 24px padding (overall); 16px gaps (internal)

Section & Grid Spacing:
├─ Page padding: 24px (all edges)
├─ Card grid gap: 16px
├─ KPI grid: repeat(auto-fit, minmax(200px, 1fr)) with 16px gap
├─ Charts grid: repeat(auto-fit, minmax(300px, 1fr)) with 16px gap
├─ Reports grid: repeat(auto-fit, minmax(250px, 1fr)) with 16px gap
└─ Row margins: 16px between major sections

Sidebar & Navigation:
├─ Sidebar width: 220px (open), 60px (collapsed)
├─ Sidebar item padding: 10px 12px
├─ Sidebar gap between items: 10px
├─ Navigation item height: ~44px (touch-friendly, 48px preferred)
└─ Sidebar bottom margin for logout: auto (space-between flex)

Responsive Breakpoints (CSS Grid):
├─ Desktop: Full 2-column layouts (content + sidebar)
├─ Tablet: Adaptive grid (auto-fit, minmax)
├─ Mobile: Single column, sidebar collapses
└─ Implementation: CSS Grid auto-responsive without hardcoded breakpoints

6. INTERACTION PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Navigation Flows (SLDS Patterns):

List → Detail → Modal:
├─ List: Click row → Detail screen (full-page)
├─ Detail: [Edit] → Modal overlay (non-disruptive)
├─ Modal: [Cancel] → Returns to Detail
│  └─ [Submit] → Closes modal, updates detail
└─ Back: [← Back] button returns to list

SLDS Pattern: Modal for non-destructive changes, full-page for primary flows

Create Workflow:
├─ List: [+ New] button → Modal form opens
├─ Form: Fill fields + [Cancel] / [Submit]
├─ Success: Modal closes, list refreshes, new item appears (ready)
├─ Error: Error message in modal (ready)
└─ SLDS Pattern: Modal forms for create/edit

Filter & Search:
├─ Search input: Top of list (in action bar)
├─ Filter buttons: Next to search (ready)
├─ Results: Instant or button-triggered (ready)
├─ Clear: X button in search (ready)
└─ SLDS Pattern: Search-first pattern

Approval Workflow:
├─ Trigger: User action (mark Converted, request gallery upload)
├─ Notification: Manager/Admin notified (bell + email)
├─ Approval: Manager reviews + [Approve]/[Reject]
├─ Outcome: Status updated, feedback to originator
└─ SLDS Pattern: Formal approval request workflow


State & Loading (SLDS Patterns):

Loading State:
├─ Spinner: Centered, animated (ready)
├─ Message: "Loading..." text (ready)
├─ Disabled: Form fields disabled during submit (ready)
├─ Button state: [Submit] → [Submitting...] → [Done] (ready)
└─ SLDS Pattern: Clear loading feedback

Empty State:
├─ Icon: Relevant illustration (ready)
├─ Message: "No items found" text
├─ Action: [Create] or [Filter] button (ready)
└─ SLDS Pattern: Actionable empty state

Error State:
├─ Icon: Error icon (AlertCircle) (ready)
├─ Message: Descriptive error text (ready)
├─ Action: [Retry] or [Go Back] button (ready)
├─ Styling: Red border or red text (ready)
└─ SLDS Pattern: Clear error messaging

Success State:
├─ Icon: Success icon (CheckCircle) (ready)
├─ Message: "Changes saved" toast message (ready)
├─ Duration: Auto-dismiss after 3-5 seconds (ready)
├─ Color: Green text/background
└─ SLDS Pattern: Confirmation feedback


7. ACCESSIBILITY (SLDS A11Y Standards)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Color Contrast (WCAG AA Standard):
├─ Text on background: 4.5:1 contrast ratio minimum
├─ Dark text (#080707) on light background (#FFFFFF): ✓ 16:1
├─ Light text (#FFFFFF) on dark background (#003366): ✓ 11:1
├─ Status badges: All meet WCAG AA standards
└─ Implementation: Consistent throughout prototype

Keyboard Navigation:
├─ Tab order: Logical flow (left-to-right, top-to-bottom)
├─ Focus indicators: Border highlight (ready)
├─ Skip links: Ready for implementation
├─ Escape key: Closes modals (ready)
├─ Enter key: Submits forms (ready)
└─ SLDS A11y: Full keyboard navigation support

Screen Reader Support:
├─ Semantic HTML: <button>, <table>, <form>, <h1>-<h6>, <input>, <label>
├─ ARIA labels: alt text on images (ready), aria-label on icons (ready)
├─ ARIA roles: Ready for role="navigation", role="main", role="dialog"
├─ Label associations: <label> for="id" (ready)
├─ List semantics: <ul>/<ol> for lists (ready)
└─ Form groups: <fieldset>/<legend> (ready)

Color Dependency:
├─ Status indicated by: Color + icon + text label (not color alone)
├─ Buttons: Include text label (not icon alone)
├─ Icons: Icon + text combination
└─ SLDS A11y: Information not conveyed by color alone

Font Size & Readability:
├─ Minimum font size: 12px (labels), 13px (body)
├─ Line height: Implicit good spacing (ready)
├─ Line length: Reasonable (not full-screen text)
├─ Font family: System sans-serif (SLDS default)
└─ SLDS A11y: Readable typography


8. RESPONSIVE DESIGN (SLDS Responsive Patterns)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Mobile-First Approach:
├─ Base styles: Mobile/tablet optimized
├─ Enhancements: Desktop larger viewports
├─ Fallbacks: Ready for older browsers
└─ SLDS Pattern: Progressive enhancement

Breakpoint Strategy:
├─ No hardcoded breakpoints (CSS Grid auto-fit)
├─ Responsive based on container size
├─ Grid templates: auto-fit with minmax

Mobile (< 640px):
├─ Sidebar: Collapses to hamburger menu icon
├─ Content: Full width
├─ Table: Horizontal scroll (ready)
├─ Modal: 90% width, max-height with scroll
├─ Buttons: Touch-friendly (48px+ recommended)
└─ Grid: Single column (auto-responsive)

Tablet (640px - 1024px):
├─ Sidebar: Visible but narrower (ready)
├─ Grid: 2-column responsive
├─ Modal: 80% width
└─ Table: Visible, may scroll

Desktop (> 1024px):
├─ Sidebar: Full 220px width
├─ Grid: 3-4 column responsive
├─ Modal: Max-width 500px
├─ Landscape: Utilizes horizontal space
└─ Table: Full display with actions
