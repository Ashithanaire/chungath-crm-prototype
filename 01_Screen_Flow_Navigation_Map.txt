SCREEN FLOW & NAVIGATION MAP

1. USER AUTHENTICATION & ENTRY POINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Login Screen
├─ Role Selection: Admin | Manager | Sales Executive | Marketing Executive
├─ Email & Password Input
└─ Redirects to → Dashboard (role-specific)

User Roles & Permissions:
─────────────────────────
• ADMIN: Cross-branch visibility, all features, approvals, system configuration
• MANAGER: Branch-level visibility, team management, lead approvals, report access
• SALES EXECUTIVE: Own leads/customers, field visits, Check-in/Out, limited mobile features
• MARKETING EXECUTIVE: Hoarding management, contractor management, route management

3. MODULE 1: LEAD MANAGEMENT FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Lead Creation:
├─ Dashboard
└─ Leads List
    ├─ [New Lead Button] → Lead Creation Modal
    │   ├─ Customer Name (Text)
    │   ├─ Phone (Tel, Mandatory)
    │   ├─ Ceremony Type (Dropdown: Marriage | Engagement | Other)
    │   ├─ Ceremony Date (Date Picker)
    │   ├─ Pincode (Text, for area auto-derivation)
    │   ├─ Referred By (Dropdown: Walk-in | Staff | Event | Other)
    │   └─ Submit → Lead Created (Status: Pending Manager Approval or Active)
    └─ → Lead List with Filtering by Status, Area

Lead Status Lifecycle:
NEW → CONTACTED → INTERESTED / NOT INTERESTED → CONVERTED / LOST / DISQUALIFIED

Lead Detail Screen:
├─ Lead Information Card (Read/Edit)
│   ├─ Customer Name
│   ├─ Phone
│   ├─ Email
│   ├─ Address
│   ├─ Pincode
│   ├─ Ceremony Type & Date
│   └─ Source of Lead
├─ Status Panel (Right Sidebar)
│   ├─ Current Status Badge
│   ├─ [Update Status Button] → Status Selection Modal
│   └─ [Record Follow-up Button] → Follow-up Modal
│       ├─ Next Status (Dropdown)
│       ├─ Visit Notes (Textarea)
│       ├─ Next Visit Date (Date Picker)
│       └─ Submit
└─ Follow-up History (Expandable)
    ├─ Status updates with timestamps
    ├─ Visit notes
    ├─ Photos (if captured)
    └─ GPS coordinates

5. MODULE 3: FIELD VISIT MANAGEMENT FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Daily Workflow (Field User):

[Start Day Button] 
├─ Auto-capture: Date, Time, GPS, Device Info
├─ Status: Daily Log Created
└─ Ready for Visits

During Field Visit:
├─ [Check-In Button]
│   ├─ Auto-capture: GPS, Timestamp
│   └─ Assign to Lead/Customer/Hoarding
├─ [Visit Details Form]
│   ├─ Visit Type (Existing Customer | New Lead | Hoarding)
│   ├─ Select Record
│   ├─ Visit Notes (Textarea)
│   ├─ Ceremony Date Confirmation (Date Picker)
│   ├─ Next Preferred Visit Date (Date Picker)
│   ├─ Status Update (Dropdown - for leads)
│   └─ [Capture Photo] → Camera (Gallery blocked by default)
├─ [Check-Out Button]
│   ├─ Auto-capture: GPS, Timestamp
│   └─ [Capture Photo] → Camera

[End Day Button]
├─ Auto-capture: Date, Time, GPS
├─ Trigger: Google Maps Distance Matrix API Call
│   ├─ Send GPS sequence: Start Day → Visit 1 → Visit 2 → ... → End Day
│   ├─ API returns: Total Road Distance (KMs)
│   └─ Store: Daily KM per Executive
└─ Daily Log Completed

Visit History per Customer:
├─ Chronological list of all visits
├─ Each visit shows:
│   ├─ Date & Times
│   ├─ GPS Coordinates (clickable map?)
│   ├─ Photos
│   ├─ Visit Notes
│   └─ Status Updates


6. MODULE 4: HOARDING MANAGEMENT FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Hoarding Creation:
Dashboard → Hoardings List → [New Hoarding Button] → Installation Modal
├─ Hoarding Name (Text)
├─ Location (Address with Landmark)
├─ Length (ft) & Width (ft) → Auto-calculate Area (Sq.Ft)
├─ Hoarding Type (Dropdown: Billboard | Poster | Digital Screen)
├─ Contractor (Lookup to Contractor Master)
├─ Annual Rate (Currency)
├─ Contract Start & End Dates
├─ Inspection Frequency (Dropdown: 30/60/90 days) → Auto-compute Next Visit Date
├─ [Capture Photo] → Camera (Installation photo - mandatory)
└─ Submit → Hoarding Created

Hoarding Detail Screen:
├─ Hoarding Information Card
│   ├─ Name, Location, Size, Area (Sq.Ft)
│   ├─ Contractor, Annual Rate, Dates
│   ├─ Installation Photo (thumbnail clickable for full view)
│   └─ Inspection Frequency
├─ Inspection Panel (Right Sidebar)
│   ├─ Next Scheduled Visit Date (Highlighted)
│   └─ [Record Inspection Button] → Board Status Modal
│       ├─ Board Status (Dropdown with sub-options)
│       │   ├─ Good Condition
│       │   ├─ Need Fix - Vegetation | Flex Damage | Competitor Hoarding
│       │   ├─ Rectified
│       │   └─ Others (requires mandatory note)
│       ├─ Inspection Date (Date Picker)
│       ├─ Notes (Textarea)
│       ├─ [Capture Photo] → Camera (Inspection photo)
│       └─ Submit
├─ Status Indicators
│   ├─ Out-of-Due-Date Flag (if inspection is late)
│   ├─ 30-Day Contract Expiry Alert (if applicable)
│   └─ Board Condition Badge (Good | Damaged | Rectified)
└─ Inspection History (Expandable)
    ├─ Each inspection with:
    │   ├─ Inspection Date & Time
    │   ├─ Board Status (color-coded)
    │   ├─ Notes
    │   ├─ GPS-tagged Photo
    │   └─ Executed By (Field User)

13. NAVIGATION PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

List View → Detail View:
├─ Click on record row → Detail screen opens (side or full-screen)
├─ [Back Button] or [← Back to List] returns to list
└─ Detail screen retains filter/sort context

Create/Edit Workflows:
├─ [+ New] / [Edit] Button → Modal dialog opens (overlay)
├─ Form validation on field exit
├─ [Cancel] dismisses modal without save
├─ [Submit] validates & saves data
└─ Success notification & modal closes

Role-based Screen Access:
├─ Each screen checks user role before rendering
├─ Unauthorized fields are hidden (FLS)
├─ Actions available only to permitted roles
└─ Sidebar items show only for user's role
